Стилизация

Точно такой же, но с квадратными ячейками

У каждого компонента есть свои исходные стили, написанные разработчиком. Они хранятся в статическом свойстве CustomCalendar.defaultStyles Это массив, который можно дополнить тегом style или добавить ссылку на таблицу стилей (вряд ли это когда-нибудь понадобится, но возможность такая есть). При необходимости, их можно дополнить или вообще полностью переписать для всего проекта. Это нужно сделать в самом начале при первом подключении компонента. Ничего не мешает добавить дополнительно классы для каждого компонента. После этого, можно инициализировать компонент. Обычно это делается в самом начале приложения.

<template id="additional-styles"><style id="extended-style"></style></template><custom-calendar data-testid="styles-customized-rect" class="rect-cells"></custom-calendar><script defer="defer">const additionalStyles=document.getElementById("additional-styles");CustomCalendar.defaultStyles.push(additionalStyles.content.cloneNode(!0)),CustomCalendar.init()</script>

Полностью перезаписанные стили

Можно перезаписать стили отдельно для уже существующего компонента. Для этого достаточно заменить тег style, в котором хранятся стили по умолчанию на свой собственный тег. Тег по умолчанию доступен по ID default-style в теневом дереве.

<custom-calendar data-testid="styles-customized-full"></custom-calendar><script>const styleTag=document.createElement("style");styleTag.textContent=fullyCustomizedCSS;const fullyCustomized=document.querySelector("[data-testid='styles-customized-full']");fullyCustomized?.shadowRoot.getElementById("default-style").replaceWith(styleTag)</script>Однако, нужно иметь в виду, что если компонент будет перевоссоздан (сработает connectedCallback), стили сбросятся к исходным. В подавляющем большинстве случаев следует предпочесть перезапись стилей по умолчанию с дополнением классами.

Альтернативно можно использовать свойство adoptedStyleSheets. Но я это не рекомендую, поскольку стили в этом случае сложнее дебажить.
