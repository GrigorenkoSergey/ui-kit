Атрибуты

  • name - имя. Необходимо, для передачи во внутренний input. Неотслеживаемое (изменение не вызовет перерисовку элемента).
  • prerendered - флаг отключения считывания шаблона и посторения html на его основе. Будет обрабатываться непосредственно переданная разметка. Неотслеживаемое.
  • open - состояние открыт/закрыт
  • value - выбранное значение в поле ввода

Базовый вариант

Поддерживает фильтрацию, выделение с клавиатуры. Для добавления первоначальных опций необходимо их добавить вручную: 

const autocomplete = document.querySelector("custom-autocomplete");
autocomplete.setOptions(["Опция-1", "Опция-2", "Опция-3", "Опция-4", "Опция-5"]);

Кол-во рендеров: 0

Кастомизированные опции

По умолчанию компонент принимает простейшие опции вида string[]. Однако, список можно настроить дополнительно. Для этого нужно переопределить метод element.renderLi. Функция принимает элемент опции, и должна вернуть элемент списка LI в виде строки с нужными атрибутами data-value, по которому будет осуществлена фильтрация и выбор. Остальные атрибуты, которые указаны в шаблоне (напр. id), тоже должны быть вычислены.

Пример: 

const withCustomizedLi = document.querySelector(".customized-li");

withCustomizedLi.renderLi = li => `\
<li part="li" data-value=${li.value}>
  ${li.value}
  <a part="link" href=${li.wiki} target="_blank" >
    ?
  </a>
</li>
`;

withCustomizedLi.setOptions(
  {
    value: "Винни-Пух",
    wiki: "https://en.wikipedia.org/wiki/Winnie-the-Pooh",
  },
  {
    value: "Пятачок",
    wiki: "https://en.wikipedia.org/wiki/Piglet_(Winnie-the-Pooh)",
  },
  {
    value: "Иа",
    wiki: "https://en.wikipedia.org/wiki/Eeyore",
  },
  {
    value: "Сова",
    wiki: "https://en.wikipedia.org/wiki/Owl_(Winnie-the-Pooh)",
  },
  {
    value: "Кролик",
    wiki: "https://en.wikipedia.org/wiki/Rabbit_(Winnie-the-Pooh)",
  },
);
