## Менеджер состояния
### API менеджера 
Изначальная реализация - в src/utils/createStore.old.js. Если в зависимости только один объект-хранилище, то все вроде бы норм.. 
Однако, как быть, когда одна и та же логика должна использоваться при изменении в нескольких хранилищах? Будет довольно неудобно.
Может быть вызывать что-то такое 
```js
const a = watch(() => store1.key1 + store2.key2 + ... + store3.key3)
```
Можно было бы в callback проверять свойства, от которых мы зависим и автоматически навешивать на них обработчики...
Можно посмотреть в сторону создания временных прокси для хранилищ, чтобы считывать свойства, полученные ловушкой get, добавлять для них обработчик и удалять прокси над этим объектом, возвращая исходное прокси. 

Скорее всего, нужно добавить ограничение, что callback ВСЕГДА должен быть запущен для определения свойств. Что там не должно быть коротких вычислений вроде
```js
const a = watch(() => trueOrFalse ? storeN.keyN : someValue);
// заменить на 
const a = watch(() => {
  const storeValue = storeN.keyN;
  return trueOrFalse ? storeValue : someValue;
})
// или
const a = watch(() => truthyAtStart || storeN.keyN);
```
